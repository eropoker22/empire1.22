## Stabilization Notes

- Public read models should use `null`, not `undefined`, for disabled fields when shared contracts say `string | null`.
- Dev-only building-card auto-open is now opt-in via `?devBuildingCard=<building>` so smoke tests and normal game boot do not inherit the last manually edited building.
- Residential cards are safe as a baseline for the next building-card passes in Industrial, Downtown, and Park zones. For those zones, keep the same checklist: data truth, disabled conditions, server/core support, legacy runtime support, and UI copy must all agree.
